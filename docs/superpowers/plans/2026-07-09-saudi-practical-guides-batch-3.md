# Saudi Practical Guides batch 3 (T54) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox syntax for tracking.

**Goal:** Add six bilingual Saudi Practical Guides (12 EN/AR HTML pages) plus category/image/noindex registration, then build, test, and open a draft PR.

**Architecture:** Reuse the T49/T50 pattern. Each article is a self-contained HTML file in `en/article/` and `ar/article/`. Category lists in `scripts/prepare.mjs` and `scripts/add-article-images.mjs` are extended. Noindex status is controlled by `editorial-review.json`. Images are generated by `add-article-images.mjs` and rendered by the build pipeline.

**Tech Stack:** Static HTML, Node.js build scripts (`prepare.mjs`, `add-article-images.mjs`, `render-og-rasters.mjs`, `audit.mjs`), git worktree.

## Global Constraints

- Work in isolated git worktree `.claude/worktrees/saudi-guides-3` on branch `content/saudi-guides-3`.
- Every article file must be 2,500–3,500 words after stripping HTML tags.
- Each article must have ≥5 FAQ pairs, with visible FAQ matching the generated `FAQPage` JSON-LD.
- Each article must have ≥3 internal links to existing doyouknow.app pages.
- Every cited external URL must resolve (`host <domain>` or curl); distinguish bot-blocking 403 from real dead links.
- Article JSON-LD is generated by `prepare.mjs`; do not hand-edit it.
- All 12 new paths go into `editorial-review.json` → `pages.government-legal-finance`.
- `npm run build` and `npm test` must exit 0 before claiming completion.
