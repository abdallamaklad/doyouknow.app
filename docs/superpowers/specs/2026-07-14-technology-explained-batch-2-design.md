# Technology Explained batch 2 (T56) — Design

## Goal

Add six new bilingual "What Is X?" AI-tool explainers (12 EN/AR HTML pages) to the `Technology Explained` category, which currently holds only 2 articles (`what-is-chatgpt`, `what-is-google-gemini`) against 8–34 articles in every other category — the thinnest, highest-opportunity category on the site.

## Content scope

| # | Slug | EN working title | Angle |
|---|---|---|---|
| 1 | `what-is-claude-ai` | What Is Claude AI? Anthropic's Assistant Explained | Positioning vs ChatGPT/Gemini, core use cases, safety framing |
| 2 | `what-is-perplexity-ai` | What Is Perplexity AI? The AI Search Engine Explained | Answer-engine model, citations, how it differs from a chatbot |
| 3 | `what-is-deepseek` | What Is DeepSeek? China's AI Model Explained | Open-weight model landscape, what "open-source AI" means practically |
| 4 | `what-is-midjourney` | What Is Midjourney? AI Image Generation Explained | How text-to-image works, Discord/web workflow, common uses |
| 5 | `what-is-grok-ai` | What Is Grok? xAI's Chatbot Explained | X/Twitter integration, positioning vs other assistants |
| 6 | `what-is-microsoft-copilot` | What Is Microsoft Copilot? Explained | Office/Windows integration, how it differs from standalone chatbots |

## Quality requirements

1. **Word count:** 2,500–3,500 words per file in both EN and AR. Self-check by stripping tags and counting before finishing.
2. **Worktree only:** All work happens in an isolated git worktree; never the shared checkout.
3. **Noindex queue:** All 12 paths added to `editorial-review.json` under `fast-changing-technology` (existing bucket already used by `what-is-chatgpt`/`what-is-google-gemini`).
4. **Volatility hedging:** This is the fastest-moving topic area on the site. Avoid hard numbers that will date quickly (pricing tiers, model version numbers, parameter counts, user-count claims). State capabilities in present-tense, avoid superlatives ("the best/largest"), and prefer framing like "as of 2026" or "at the time of writing" where a claim could plausibly change within months. Every factual claim must trace to a source that is the vendor's own site or a reputable tech outlet — no forums, no unverified benchmarks.
5. **Source verification:** Every cited URL must resolve. Use `host <domain>` at minimum; if a vendor site returns 403 due to bot-blocking, cross-reference via search rather than assuming it is dead.
6. **FAQPage JSON-LD:** Hand-sync visible FAQ markup (5+ questions) with the `FAQPage` schema. Article JSON-LD is auto-generated by `prepare.mjs` and must not be edited by hand.
7. **Internal links:** ≥3 internal links per article, including links to the two existing `technology` articles and to each other where the comparison is natural (e.g., Claude ↔ ChatGPT, Midjourney ↔ other AI tools).
8. **Build/test:** `npm run build && npm test` must pass with real exit code 0.

## Architecture

- **Branch:** `content/technology-batch-2` from current `main`.
- **Worktree:** `.claude/worktrees/technology-batch-2`.
- **Content files:** `en/article/<slug>.html` and `ar/article/<slug>.html` for each of the six slugs.
- **Category registration:** Append each slug to the EN `technology` and AR `technology` category lists in `scripts/prepare.mjs` (both currently `['what-is-chatgpt','what-is-google-gemini']`).
- **Image registration:** Add a `technology` category to `scripts/add-article-images.mjs` if not already present (check first — `dubai`/`guides`/`business`/`saudi`/`vision-2030`/`islamic` categories exist there already; `technology` may need to be added from scratch, matching the palette/label/icon pattern used for `business` in T46).
- **Featured/OG images:** Generate SVGs and 1200×630 PNGs for all 12 articles using `add-article-images.mjs` and `render-og-rasters.mjs`.
- **Noindex:** Add all 12 article paths to `editorial-review.json` → `pages.fast-changing-technology`.

## Approach

Follow the same workflow proven in T49/T50/T54:

1. Create the worktree and verify a clean baseline (`npm test`).
2. Draft the six EN articles to the target word count, including editorial note, section headings, sources, FAQ, and related-articles cards.
3. Draft the six AR translations, preserving structure and internal links.
4. Register the new slugs in `scripts/prepare.mjs`, and add/extend the `technology` category in `scripts/add-article-images.mjs`.
5. Add the 12 paths to `editorial-review.json`.
6. Run `npm run build` to generate images, schemas, sitemaps, feeds, and category pages.
7. Verify every cited source URL resolves.
8. Run `npm test` and confirm exit code 0.
9. Commit and push the branch.
10. Open a draft PR against `main` summarizing the batch, source verification, volatility-hedging choices, and noindex status.

## Execution preference

Use subagent-driven development: dispatch one subagent per article pair (EN + AR) so each can be drafted, counted, and reviewed independently, while the coordinator handles category/image registration, source verification, build/test, and final commit.
