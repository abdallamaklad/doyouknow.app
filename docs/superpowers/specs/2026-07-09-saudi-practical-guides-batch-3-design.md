# Saudi Practical Guides batch 3 (T54) — Design

## Goal

Add six new bilingual Saudi Practical Guides (12 EN/AR HTML pages) covering GOSI, foreign-driving-license conversion, Tawakkalna, bank-account types, labor-law basics, and customs/import rules for residents.

## Content scope

| # | Slug | EN working title | Angle |
|---|---|---|---|
| 1 | `saudi-gosi-guide` | GOSI in Saudi Arabia: Contributions, Benefits, and Registration | Social insurance basics for employees and employers |
| 2 | `saudi-driving-license-transfer` | Converting a Foreign License to a Saudi Driving License | Eligible countries, test requirements, costs |
| 3 | `tawakkalna-guide` | Tawakkalna App: What It Does and Why Residents Need It | Health/ID integration and common use cases |
| 4 | `saudi-bank-account-types` | Bank Accounts in Saudi Arabia: Types, Requirements, Fees | SAMA-regulated options, resident vs non-resident |
| 5 | `saudi-labor-law-basics` | Saudi Labor Law Basics: Contracts, Leave, End of Service | Key employee rights, GOSI-adjacent but distinct |
| 6 | `saudi-customs-import-guide` | Bringing Goods Into Saudi Arabia: Customs Rules for Residents | Personal shipments, duty-free limits, banned items |

## Quality requirements

1. **Word count:** 2,500–3,500 words per file in both EN and AR. Self-check by stripping tags and counting before finishing.
2. **Worktree only:** All work happens in an isolated git worktree; never the shared checkout.
3. **Noindex queue:** All 12 paths added to `editorial-review.json` under `government-legal-finance`.
4. **Source verification:** Every cited URL must resolve. Use `host <domain>` at minimum; if a gov site returns 403 due to bot-blocking, cross-reference via search rather than assuming it is dead.
5. **FAQPage JSON-LD:** Hand-sync visible FAQ markup (5+ questions) with the `FAQPage` schema. Article JSON-LD is auto-generated by `prepare.mjs` and must not be edited by hand.
6. **Internal links:** ≥3 internal links per article.
7. **Build/test:** `npm run build && npm test` must pass with real exit code 0.

## Architecture

- **Branch:** `content/saudi-guides-3` from current `main` (after T53 merges landed).
- **Worktree:** `.claude/worktrees/saudi-guides-3`.
- **Content files:** `en/article/<slug>.html` and `ar/article/<slug>.html` for each of the six slugs.
- **Category registration:** Append each slug to the EN `saudi-guides` and AR `guides` category lists in `scripts/prepare.mjs`.
- **Image registration:** The `saudi-guides` category already exists in `scripts/add-article-images.mjs`; extend its file list to include the six new slugs.
- **Featured/OG images:** Generate SVGs and 1200×630 PNGs for all 12 articles using existing `add-article-images.mjs` and `render-og-rasters.mjs`.
- **Noindex:** Add all 12 article paths to `editorial-review.json` → `pages.government-legal-finance`.

## Approach

Follow the same workflow proven in T49/T50:

1. Create the worktree and verify a clean baseline (`npm test`).
2. Draft the six EN articles to the target word count, including editorial note, section headings, sources, FAQ, and related-articles cards.
3. Draft the six AR translations, preserving structure and internal links.
4. Register the new slugs in `scripts/prepare.mjs` and `scripts/add-article-images.mjs`.
5. Add the 12 paths to `editorial-review.json`.
6. Run `npm run build` to generate images, schemas, sitemaps, feeds, and category pages.
7. Verify every cited source URL resolves.
8. Run `npm test` and confirm exit code 0.
9. Commit and push the branch.
10. Open a draft PR against `main` summarizing the batch, source verification, and noindex status.

## Execution preference

Use subagent-driven development: dispatch one subagent per article pair (EN + AR) so each can be drafted, counted, and reviewed independently, while I coordinate category registration, image generation, source verification, build/test, and final commit.
