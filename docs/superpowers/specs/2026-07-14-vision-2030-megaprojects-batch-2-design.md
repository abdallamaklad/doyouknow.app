# Saudi Vision 2030 & Megaprojects batch 2 (T57) — Design

## Goal

Add six new bilingual explainers (12 EN/AR HTML pages) to the `Saudi Vision 2030 & Megaprojects` category, extending it from 8 to 14 articles with giga-projects and national initiatives not yet covered (current set: `neom-city-facts`, `saudi-vision-2030-guide`, `what-is-neom`, `the-line-neom`, `red-sea-project-saudi`, `kingdom-tower-riyadh`, `riyadh-season`, `qiddiya-saudi-arabia`).

## Quality requirements

1. **Word count:** 2,500–3,500 words per file in both EN and AR. Self-check by stripping tags and counting before finishing.
2. **Worktree only:** All work happens in an isolated git worktree; never the shared checkout.
3. **Noindex queue:** All 12 paths added to `editorial-review.json` under `government-legal-finance` (giga-project/economic-policy content, consistent with how `saudi-vision-2030-guide` and NEOM articles are already classified — verify against current file before assuming the bucket name).
4. **Source verification:** Every cited URL must resolve. Prefer official giga-project sites (neom.com, roshn.sa, pif.gov.sa, rsg.sa) and PIF/government press releases over secondary news aggregators. If a government/giga-project site returns 403 due to bot-blocking, cross-reference via search rather than assuming it is dead.
5. **Timeline hedging:** Giga-projects regularly slip announced completion dates. State target dates as "announced for" or "planned for," not as settled fact, and avoid claiming a phase is complete unless independently confirmed.
6. **FAQPage JSON-LD:** Hand-sync visible FAQ markup (5+ questions) with the `FAQPage` schema. Article JSON-LD is auto-generated by `prepare.mjs` and must not be edited by hand.
7. **Internal links:** ≥3 internal links per article, including links to the existing `vision-2030` cluster (especially `saudi-vision-2030-guide` as the hub article) and to `saudi-pif-explained` from the other five giga-project articles where PIF funding is mentioned.
8. **Build/test:** `npm run build && npm test` must pass with real exit code 0.

## Architecture

- **Branch:** `content/vision-2030-batch-2` from current `main`.
- **Worktree:** `.claude/worktrees/vision-2030-batch-2`.
- **Content files:** `en/article/<slug>.html` and `ar/article/<slug>.html` for each of the six slugs.
- **Category registration:** Append each slug to the EN `vision-2030` and AR `vision-2030` category lists in `scripts/prepare.mjs`.
- **Image registration:** The `vision-2030` category already exists in `scripts/add-article-images.mjs`; extend its file list to include the six new slugs.
- **Featured/OG images:** Generate SVGs and 1200×630 PNGs for all 12 articles using `add-article-images.mjs` and `render-og-rasters.mjs`.
- **Noindex:** Add all 12 article paths to `editorial-review.json` (confirm correct bucket key against the current file — do not assume without checking).

## Approach

Follow the same workflow proven in T49/T50/T54:

1. Create the worktree and verify a clean baseline (`npm test`).
2. Draft the six EN articles to the target word count, including editorial note, section headings, sources, FAQ, and related-articles cards.
3. Draft the six AR translations, preserving structure and internal links.
4. Register the new slugs in `scripts/prepare.mjs` and extend the `vision-2030` category in `scripts/add-article-images.mjs`.
5. Add the 12 paths to `editorial-review.json` under the correct bucket.
6. Run `npm run build` to generate images, schemas, sitemaps, feeds, and category pages.
7. Verify every cited source URL resolves.
8. Run `npm test` and confirm exit code 0.
9. Commit and push the branch.
10. Open a draft PR against `main` summarizing the batch, source verification, timeline-hedging choices, and noindex status.

## Execution preference

Use subagent-driven development: dispatch one subagent per article pair (EN + AR) so each can be drafted, counted, and reviewed independently, while the coordinator handles category/image registration, source verification, build/test, and final commit.
